# Update ONBOARDING.md

## Overview

Update the existing root `ONBOARDING.md` to reflect the current `commands/` and `rules/` without fully rewriting. Preserve structure and tone; add or adjust only what has changed.

## Steps

1. **Read current state**
    - Read current `ONBOARDING.md` in the repository root
    - List all files in `commands/` and `rules/` to see current commands and rules
2. **Identify gaps**
    - Missing commands or rules in the doc
    - Outdated sections (e.g. renamed or removed commands/rules)
    - New sections needed (only if the plan’s section list has grown)
3. **Edit in place**
    - Add or update sections for new or changed commands/rules
    - Adjust "Repo structure" or "First steps" only if the repo layout has changed
    - Leave unchanged sections intact; do not rewrite for style only
4. **Consistency**
    - Keep the same section order and style as in the create-onboarding command so both commands produce a consistent doc
    - Required section order: Project overview, Prerequisites, Repo structure, Using commands, Using rules, First steps, Getting help

## Update ONBOARDING.md Checklist

- [ ] All current commands and rules are reflected in the doc
- [ ] No obsolete command or rule names remain
- [ ] Section order matches create-onboarding command
- [ ] Edits are minimal and targeted; tone preserved
